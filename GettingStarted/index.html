         GETTING STARTED WITH GIT AND GITHUB

   

Yes, for getting started with Git and GitHub, you should know what it means, Right?

Before that, let's say you and your friend are working on a team project and you have split the project into two. You have begun working with yours and if you are someone like me, you have made copies of the same project saved in different versions in your workspace named Rough1, Rough2 etc., You are almost on the verge of completing your part. Now what? You must edit stuff from your rough versions and get the complete draft or at least that's what I did before learning about Git and GitHub.

How is GitHub helping?

>In the above case, my life would have been much easier if I had used a version control system - Git to keep track of all the work I've been doing with the project and stored a copy of my projects using the service - GitHub in a repository (GitHub folder)

To understand Git and GitHub without any code, there is a project on GitHub at https://guides.github.com/activities/hello-world/ which explains the terminologies you should know to work with GitHub.

 After successfully completing the steps involved, I learnt, that by creating a repository for my project, I would have started with version 1 and instead of creating separate versions 2 and 3, I would have worked with creating a branch with version 1 and added the changes to my branch using commit and finally once I have decided to compile into a final draft, I would have used merge to save it to version 1. Also, by using GitHub, I would have combined the second part of the project that my friend worked on by sending in a pull request to her repository.

Still don't have an account on GitHub? It takes less than 5 minutes to sign up at https://github.com/
